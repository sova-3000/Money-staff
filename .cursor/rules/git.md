# Правила работы с Git

## Настройка автора коммитов

**КРИТИЧЕСКИЙ МОМЕНТ**: Перед началом работы с git в этом проекте всегда проверяй и устанавливай правильные настройки автора коммитов.

### Требуемые настройки для этого проекта:
```bash
git config user.name "sova-3000"
git config user.email "premiumservice23@gmail.com"
```

### Проверка настроек:
```bash
git config user.name
git config user.email
```

### Если настройки неправильные:
- **НИКОГДА** не создавай коммиты с дефолтными значениями (`User Name`, `user@example.com`)
- Всегда проверяй настройки перед первым коммитом в сессии
- Если случайно создал коммит с неправильным автором - исправь через `git commit --amend --author="sova-3000 <premiumservice23@gmail.com>"` или через rebase

### Глобальная настройка (опционально, но рекомендуется):
Для всех проектов на этой машине:
```bash
git config --global user.name "sova-3000"
git config --global user.email "premiumservice23@gmail.com"
```

## Работа с коммитами

### Перед коммитом:
1. Проверить `git config user.name` и `git config user.email`
2. Проверить изменения через `git status` и `git diff`
3. Убедиться, что коммит содержит осмысленное сообщение

### Формат сообщений коммитов:
- Используй русский язык для сообщений коммитов
- Для checkpoint-коммитов: `checkpoint: описание состояния`
- Для триггеров: `trigger: описание действия`
- Для фич: `feat: описание`
- Для багфиксов: `fix: описание`

## Работа с ветками

- Основная ветка: `main`
- Все изменения пушатся в `origin/main`
- Для экспериментальных изменений создавай отдельные ветки

## Force push

**ВНИМАНИЕ**: Force push допустим только в следующих случаях:
- Исправление автора коммитов (после rebase)
- Исправление истории после согласования
- Использовать `--force-with-lease` вместо `--force` для безопасности

## Точки сохранения (checkpoints)

При создании важных точек сохранения:
1. Создай коммит с префиксом `checkpoint:`
2. Создай тег для легкого отката: `git tag -a checkpoint-name-v1 -m "Описание"`
3. Запушь и коммит, и тег: `git push origin main && git push origin checkpoint-name-v1`

## Интеграция с Vercel

- Vercel автоматически деплоит при пуше в `main`
- Если деплой не запустился автоматически - создать пустой коммит-триггер
- После изменений git-настроек проверить, что коммиты идут от правильного автора
